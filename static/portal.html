这版用户端已切到“账号模式优先”

管理员敏感操作仍留在 `/admin`，前台只保留用户能看、能用、能自助处理的能力。

01

终端用户注册 / 登录

主路径新增账号体系，页面会明确显示注册位置、是否已注册、最后登录和当前状态。

02

CDK 兑换与成功提示

用户输入兑换码即可领取打码次数，兑换成功后会立即显示新增次数和最近兑换结果。

03

次数余额与成功统计

展示剩余次数、已用次数、成功次数、失败次数，并明确说明“失败不扣次”。

04

用户态 Solve 调试

主按钮优先调用 `/api/portal/user/*`，不再把 API Key 作为前台默认主入口。

05

Session 工具箱

继续支持 finish / error，会自动回填最近 session_id，方便用户闭环联调。

06

高级兼容模式

仍保留 API Key 连接方式，作为高级模式 / 兼容模式，不影响历史接入方。
