接入说明

子节点主页只保留运行状态展示，用于让部署人员确认当前实例角色、浏览器容量和健康状态是否正常。

节点职责

  • 执行本地 Playwright + Chromium 打码。
  • 向主节点注册并持续上报心跳。
  • 接收 routed session 的 finish / error 回调。

排查入口

  • /api/v1/health 查看实例存活状态。
  • /admin 查看配置、日志和节点执行信息。
  • /api/portal/summary 查看公开运行摘要。

如果这里显示的角色不是 subnode，请检查 FCS_CLUSTER_ROLE 或 data/setting.toml。
